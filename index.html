Getting Started

Install & Launch

Bandito runs on Python 3.10+ and is designed for Kali Linux. For buffer overflow modules, Metasploit must be installed. All DVWA modules require a running DVWA instance.

01
Clone the repository
Pull the latest version from GitHub into your working directory.
git clone https://github.com/BradJSutt/bandito-framework.git
cd bandito-framework
02
Install dependencies
Install required Python packages. A virtual environment is recommended.
pip3 install -r requirements.txt
03
Launch the framework
Run the entry point. Bandito auto-discovers and loads all modules under modules/.
python3 bandito.py
04
Requirements
Make sure your environment has what Bandito needs before running.
Python 3.10+   # required
Kali Linux    # recommended
Metasploit    # BoF modules only
DVWA running  # DVWA modules

What's Included

Available Modules

Six attack modules covering the major vulnerability classes. Each auto-loads at startup and follows the same use / set / run interface.

command_injection
Active
rce
Command injection to full RCE. Opens an interactive shell via injection, then optionally uploads a PHP webshell or auto-deploys a reverse shell with a listener.
run  — inject & open shell
upload_webshell  — drop shell.php
upload_revshell  — reverse shell + nc listener
shell  — interactive command loop
xss
Active
xss_reflected
Reflected XSS payload generation with an HTTP catch server. Opens the DVWA XSS page automatically and generates a payload to paste into the form field.
setup  — start catch server, open page
generate  — print payload to clipboard
xss
Active
xss_stored
Stored XSS cookie stealer targeting the DVWA guestbook. Injects a persistent payload that exfiltrates session cookies to your catch server.
setup  — start server, open stored XSS page
generate  — print guestbook payload
sqli
Active
sqli
Full SQL injection attack chain: logs in, dumps user hashes, then pipes them straight into Hashcat for cracking. Results printed to terminal.
run  — login → dump hashes → crack
buffer_overflow
Active
bof_analysis
Guided buffer overflow analysis chain against Benji's Snack Vault. Steps through fuzzing → pattern creation → offset identification → bad character detection → EIP control verification. Saves results to JSON.
run  — full guided analysis chain
saves → analysis_results.json
buffer_overflow
Active
bof_exploit
Automated exploitation using analysis results. Generates a Meterpreter payload via msfvenom, constructs the exploit buffer, and fires for a shell.
run  — generate payload + fire exploit
reads ← analysis_results.json

Extending Bandito

Build Your Own Module

Bandito is fully modular. Drop any .py file under modules/ and it auto-loads at startup — no config needed.

modules/my_module/exploit.py
# 1. Import the right base class
from base_module import BaseModule
# For DVWA targets, use DVWABase instead:
# from dvwa_base import DVWABase
class MyModule(BaseModule):
    def __init__(self):
        super().__init__()
        # 2. Set module identity
        self.name = "my_module"
        self.description = "Does something cool"
        # 3. Define your options
        self.options = {
            "TARGET": {
                "value": "",
                "required": True,
                "description": "Target IP"
            },
        }
    # 4. Implement run()
    def run(self):
        target = self.options['TARGET']['value']
        print(f"[+] Running against {target}")
        # your logic here
01
File structure
Create any .py file anywhere under modules/. Subdirectories become the module namespace.
modules/
  my_category/
    my_exploit.py  ← auto-loaded as
                      my_category.my_exploit
02
Base class — which to use
Pick your parent class based on your target:
BaseModule   ← general purpose
DVWABase     ← auto DVWA login + CSRF
03
Optional overrides
Customize behaviour by overriding these methods in your class:
show_help()       ← custom help text
handle_command()  ← extra CLI commands
run()             ← required: main logic
04
Load it
Restart Bandito. Your module appears automatically in show modules and is available via use.
python3 bandito.py
[+] Loaded module: my_category.my_exploit
